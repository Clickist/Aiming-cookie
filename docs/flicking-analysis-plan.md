# Flicking 分析方案

## 目标

扩展 Tension-Aware-Aim-Analyzer 支持 flicking（甩枪）场景分析。当前仅支持 tracking（跟枪）。

## 核心卡点

从纯录像无法检测 miss 的点击——画面零反馈，无法区分"还没点"和"点了但 miss"。纯录像方案有幸存者偏差，只能分析命中样本。

## 选定方案：KovaaK's 场景数据（方案B）

直接读取 KovaaK's 的内部数据（scenario data / 回放文件），获取：
- 每次射击的时间戳
- 命中/miss 状态
- 射击时的准星坐标
- 目标位置

### 优势
- 最准确，无时间对齐误差
- 同时覆盖命中和 miss
- 简化整个系统——不需要 CV 检测目标位置，不需要 CSRT 追踪
- 可以直接算反应时间、点击精度、miss 率等完整指标

### 需要调研
1. KovaaK's scenario data 的数据格式（JSON? CSV? 二进制?）
2. 数据文件的存储位置
3. 包含哪些字段（射击时间、命中/miss、坐标、目标信息等）
4. 数据获取方式（文件读取? API? 回放解析?）
5. 不同版本的数据格式兼容性

## Flicking 分析指标（待定，取决于数据可用性）

| 指标 | 需要的数据 | 意义 |
|---|---|---|
| 反应时间 | 目标出现时间 + 第一次点击时间 | 视觉反应速度 |
| 点击精度 | 点击时准星位置 vs 目标位置 | flick 准确度 |
| 过冲/欠冲 | 准星速度曲线 + 最终位置 | 握力控制 |
| miss 率 | 命中/miss 计数 | 整体精度 |
| flick 速度 | 目标出现到命中的时间 | 执行速度 |
| 速度曲线 | 准星在 flick 区间的运动 | 平滑度、两段式修正检测 |

## 实施路径

### Phase 0：搞明白 KovaaK's 怎么产出数据（当前卡点）
1. KovaaK's 有没有内置的数据记录/导出功能？在哪里开启？
2. 记录的数据存在哪？文件格式是什么？
3. 数据包含哪些字段？（射击时间、命中/miss、坐标、目标信息等）
4. 不同场景类型（tracking vs flicking）的数据格式一样吗？
5. 有没有第三方工具/插件可以增强数据导出？

### Phase 1：数据读取 prototype
1. 写一个最小脚本读取 Phase 0 发现的数据文件
2. 确认字段含义和数据质量

### Phase 2：指标设计 + 集成
1. 设计 flicking 场景的指标模型
2. 集成到现有系统（可能需要新的分析管线 + dashboard 标签页）
